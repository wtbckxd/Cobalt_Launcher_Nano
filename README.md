### 🚀 Cobalt Launcher Nano

Лёгкий, быстрый и мощный лаунчер для Minecraft на Python.
Устанавливайте ванильные версии, Forge, Fabric — всё в отдельные профили, как в официальном лаунчере.
Никаких сторонних библиотек для установки версий — только прямые API Mojang, Fabric и Forge и тд.

### ✨ Возможности

    ✅ Установка любых версий Minecraft (альфа, бета, снапшоты, релизы)
    🔧 Автоматическая установка Forge и Fabric как отдельных версий
    🧩 Поддержка модов – поиск и установка с Modrinth, избранное
    👤 Управление оффлайн-аккаунтами – добавление, переименование, удаление
    ⚙️ Настройка Java – автоматическая загрузка (8/11/17/21), ручной путь, аргументы JVM
    📁 Быстрый доступ к папкам – моды, ресурспаки, миры, скриншоты, конфиги
    💾 Резервное копирование миров, ресурспаков, конфигов и модов
    🔌 Плагины – расширяйте функциональность своими скриптами
    📝 Заметки – встроенный блокнот для важной информации
    🖥️ Кроссплатформенность – Windows, Linux, macOS

### 📦 Установка

1) Скачайте лаунчер
    Скачайте Cobalt_Launcher_Nano.exe
    Распакуйте в любую папку (например, C:\CobaltLauncher)

2) Запустите лаунчер

3) После запуска вы увидите баннер и краткую информацию о системе.
Введите помощь для полного списка команд.
4) **Небольшая инструкция**
    акк
    → 1 (оффлайн)
    → введите любое имя (например, `Steve`)

    Установите версию Minecraft
    установить 1.20.1

    С модлоадером или без:
    установить 1.20.1 forge
    запуск

**Готово! 🎉**

### ⚙️ Настройка Java

  установить джава	Скачать и установить Java (выбор версии 8/11/17/21)

  память 4	Выделить 4 ГБ RAM (меняет -Xmx и -Xms)

  джава	Указать путь к Java вручную (или Enter для сброса)

  джава версии 1.8.9 C:\путь\java.exe	Привязать Java к конкретной версии Minecraft
  
  арг	Настроить общие аргументы JVM (например, -XX:+UseG1GC)

  арг версии 1.20.1 -Xmx6G	Аргументы для конкретной версии

  отдельные папки	Включить/выключить отдельные .minecraft для каждой версии

📂 Структура данных

Все файлы лаунчера хранятся в папке C:\cobalt_launcher (Windows) или ~/cobalt_launcher (Linux/macOS).
text

cobalt_launcher/
├── minecraft/               # Главная папка .minecraft
│   ├── versions/            # Установленные версии (каждая в своей папке)
│   ├── mods/                # Моды
│   ├── saves/               # Миры
│   ├── resourcepacks/       # Ресурспаки
│   ├── shaderpacks/         # Шейдеры
│   ├── config/              # Конфиги модов
│   ├── schematics/          # Схемы
│   ├── logs/                # Логи игры
│   ├── crashes/             # Краш-репорты
│   └── launcher_profiles.json # Аккаунты (оффлайн)
├── java/                    # Автоматически установленные Java
│   ├── java_8/
│   ├── java_11/
│   ├── java_17/
│   └── java_21/
├── plugins/                 # Плагины лаунчера (.py файлы)
├── config.json              # Настройки лаунчера
├── notes.txt                # Ваши заметки
└── mods_favorites.json      # Избранные моды

🛠️ Для разработчиков
Плагины

Положите Python-файл в папку plugins, и он автоматически загрузится при запуске.

Пример простого плагина:
python

def register_plugin(api):
    api.register_command("hello", lambda args: print("Hello from plugin!"))
    api.register_banner_hook(lambda banner: banner + "Custom banner!\n")

Доступные методы PluginAPI:

    register_command(name, func, hidden=False) – добавить команду

    register_banner_hook(func) – изменить баннер

    register_info_hook(func) – изменить блок информации

    get_data_dir() – путь к папке лаунчера

    get_plugins_dir() – путь к папке плагинов

### Сборка в EXE (Windows)
bash

pip install pyinstaller
pyinstaller --onefile --name CobaltLauncher --add-data "constants.py;." --add-data "utils.py;." --add-data "config_manager.py;." --add-data "accounts.py;." --add-data "java_manager.py;." --add-data "version_manager.py;." --add-data "mods_manager.py;." --add-data "plugin_api.py;." --add-data "ui.py;." main.py

Зависимости (только для плагинов и модов)

Основной код использует только стандартные модули Python (urllib, json, os, zipfile, subprocess и т.д.).
Дополнительные библиотеки нужны только для:

    colored – цветной вывод в консоль

    keyboard – глобальная горячая клавиша (Windows)

    nbtlib – добавление серверов в servers.dat (опционально)

### ❓ Частые проблемы и решения
1. Ошибка ClassCastException при запуске 1.8.9 / 1.12.2

Причина: старые версии требуют Java 8, а у вас Java 17+.
Решение:
text

установить джава
→ 1 (Java 8)

Лаунчер автоматически подхватит Java 8 для этих версий.
2. Не устанавливается Forge

    Убедитесь, что версия Minecraft существует и поддерживает Forge.

    Если автоматическая установка не удалась, лаунчер предложит открыть страницу Forge в браузере.

    Некоторые очень старые версии (1.5.2 и ниже) не поддерживают автоматическую установку.

3. Ошибка No module named 'colored'

Выполните:
bash

pip install colored

4. Лаунчер не видит установленные версии

Проверьте папку cobalt_launcher/minecraft/versions – там должны быть папки с .json и .jar файлами.
5. Не запускается мод с зависимостями

Установите нужные API через лаунчер:
text

fabricapi

или
text

forgeapi

(Forge API встроен в сам Forge).
